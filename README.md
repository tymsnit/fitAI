# FitAI

**FitAI** — це вебзастосунок для формування персональних тренувальних планів та рекомендацій на основі профілю користувача, історії виконання тренувань і суб’єктивної оцінки складності.

Проєкт розроблено як дипломний вебзастосунок із використанням стеку:

* React;
* Node.js;
* Express;
* PostgreSQL;
* REST API;
* JWT-авторизація;
* рекомендаційна логіка на основі правил.

## 1. Основна ідея проєкту

FitAI дозволяє користувачу:

* зареєструватися;
* увійти в систему;
* заповнити особистий профіль;
* переглядати базу вправ;
* згенерувати персональний тренувальний план;
* позначати тренування як виконані;
* переглядати історію тренувань;
* отримувати персональні AI-рекомендації.

AI-рекомендації у MVP реалізовані як система рекомендацій на основі правил. Система аналізує профіль користувача, активний план, кількість виконаних тренувань, відсоток виконання та суб’єктивну складність тренувань.

## 4. Налаштування бази даних

Потрібно створити базу даних PostgreSQL:

```sql
CREATE DATABASE fitai;
```

Після цього виконати SQL-файли:

```bash
psql -U postgres -d fitai -f database/schema.sql
psql -U postgres -d fitai -f database/seed.sql
```

Або виконати вміст файлів `schema.sql` та `seed.sql` через pgAdmin Query Tool.

## 5. Налаштування backend

Перейти в папку backend:

```bash
cd server
```

Встановити залежності:

```bash
npm install
```

Створити файл `.env` у папці `server`:

```env
PORT=5000

DB_HOST=localhost
DB_PORT=5432
DB_NAME=fitai
DB_USER=postgres
DB_PASSWORD=your_password

JWT_SECRET=fitai_secret_key
```

Запустити backend:

```bash
npm run dev
```

Backend буде доступний за адресою:

```text
http://localhost:5000
```

## 9. Основний демонстраційний сценарій

1. Відкрити головну сторінку FitAI.
2. Зареєструвати нового користувача.
3. Увійти в систему.
4. Заповнити профіль користувача:

   * ім’я;
   * вік;
   * стать;
   * зріст;
   * вага;
   * рівень підготовки;
   * ціль;
   * кількість тренувань на тиждень.
5. Перейти до бази вправ.
6. Показати вправи та фільтри.
7. Перейти до сторінки “Мій план”.
8. Згенерувати персональний тренувальний план.
9. Переглянути тренування по днях.
10. Позначити одне тренування як виконане.
11. Перейти до історії тренувань.
12. Переглянути запис про виконане тренування.
13. Перейти до AI-рекомендацій.
14. Показати рекомендації та пояснення логіки.
15. Повернутися до Dashboard і показати зведення.

## 10. Логіка AI-рекомендацій

У межах MVP FitAI використовує рекомендаційну систему на основі правил.

Система аналізує:

* рівень підготовки користувача;
* ціль користувача;
* кількість тренувань у плані;
* кількість виконаних тренувань;
* відсоток виконання плану;
* суб’єктивну оцінку складності тренувань.

Приклади правил:

* якщо користувач виконав менше 50% плану, система рекомендує зменшити навантаження;
* якщо користувач виконав понад 80% плану, система може рекомендувати поступове підвищення складності;
* якщо тренування часто позначаються як `too_hard`, система рекомендує зменшити складність;
* якщо мета користувача — `weight_loss`, система рекомендує регулярність і кардіонавантаження;
* якщо мета користувача — `muscle_gain`, система рекомендує силові вправи та поступове збільшення навантаження.

## 11. Очікуваний результат

У результаті реалізації проєкту створено повноцінний MVP вебзастосунку FitAI, який демонструє:

* роботу з користувачами;
* авторизацію;
* збереження даних у PostgreSQL;
* формування персонального тренувального плану;
* збереження історії виконання;
* формування персональних рекомендацій;
* взаємодію React frontend з Node.js backend через REST API.

## 12. Обмеження MVP

FitAI не є медичним сервісом і не замінює консультацію лікаря або професійного тренера. Рекомендації мають демонстраційний характер і використовуються для показу алгоритмічної персоналізації в межах дипломного проєкту.
