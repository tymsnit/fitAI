# FitAI

**FitAI** — це вебплатформа для формування персональних тренувальних планів, ведення історії тренувань, обліку харчування та отримання рекомендацій на основі профілю користувача, тренувальної активності й добових показників харчування.

Проєкт розроблено як дипломний вебзастосунок із використанням стеку:

* React;
* Vite;
* Node.js;
* Express;
* PostgreSQL;
* REST API;
* JWT-авторизація;
* rule-based expert system;
* інтеграція із зовнішнім OpenAI API для текстового AI-узагальнення рекомендацій.

FitAI не реалізує власну нейронну мережу. Алгоритмічна персоналізація у застосунку базується на пояснюваній системі правил, а OpenAI API використовується як зовнішній AI-сервіс для генерації текстового підсумку рекомендацій.

## 1. Основна ідея проєкту

FitAI дозволяє користувачу:

* зареєструватися;
* увійти в систему;
* працювати з особистим профілем;
* переглядати базу вправ;
* фільтрувати вправи;
* генерувати персональний тренувальний план;
* переглядати тренувальний план по днях;
* позначати тренування як виконані;
* переглядати історію виконаних тренувань;
* вести журнал харчування;
* фіксувати калорії, білки, жири, вуглеводи та воду;
* формувати або вручну редагувати добові цілі харчування;
* переглядати підсумок харчування за день;
* працювати з Dashboard, який об’єднує профіль, тренування, харчування та рекомендації;
* отримувати персональні рекомендації, сформовані rule-based expert system;
* отримувати текстове AI-узагальнення рекомендацій через зовнішній OpenAI API.

Рекомендаційний модуль FitAI реалізовано як пояснювану експертну систему на основі правил. Система аналізує дані профілю користувача, активний тренувальний план, історію виконання тренувань, рівень виконання плану, суб’єктивну складність тренувань, а також базові показники харчування.

Кожна рекомендація може містити службові поля, які пояснюють її походження:

* `ruleId` — ідентифікатор правила;
* `type` — тип рекомендації;
* `priority` — пріоритет;
* `explanation` — пояснення логіки формування рекомендації.

## 4. База даних

База даних FitAI побудована на PostgreSQL.

Основні сутності бази даних:

* `users`;
* `user_profiles`;
* `exercises`;
* `workout_plans`;
* `workouts`;
* `workout_exercises`;
* `workout_logs`;
* `nutrition_targets`;
* `nutrition_logs`;
* `recommendations`.

Для створення бази даних потрібно виконати:

```sql
CREATE DATABASE fitai;
```

Після цього потрібно виконати SQL-файли:

```bash
psql -U postgres -d fitai -f database/schema.sql
psql -U postgres -d fitai -f database/seed.sql
```

Альтернативно вміст файлів `schema.sql` та `seed.sql` можна виконати через pgAdmin Query Tool.

## 5. Налаштування backend

Перейти в папку backend:

```bash
cd server
```

Встановити залежності:

```bash
npm install
```

У папці `server` потрібно створити файл `.env` на основі прикладу `.env.example`.

Приклад файлу `.env.example`:

```env
PORT=5000

DB_HOST=localhost
DB_PORT=5432
DB_NAME=fitai
DB_USER=postgres
DB_PASSWORD=your_password

JWT_SECRET=change_this_secret

OPENAI_API_KEY=your_openai_api_key_here
```

Файл `.env` не повинен потрапляти до Git-репозиторію, оскільки він містить локальні налаштування та секретні значення.

`OPENAI_API_KEY` використовується тільки для підключення до зовнішнього OpenAI API. У README, прикладах коду та публічному репозиторії не можна вказувати реальний API-ключ.

Запустити backend у режимі розробки:

```bash
npm run dev
```

Backend буде доступний за адресою:

```text
http://localhost:5000
```

Endpoint `/api/ai-recommendations/summary` використовує зовнішній OpenAI API для формування текстового узагальнення рекомендацій. OpenAI API не замінює власну бізнес-логіку FitAI, а використовується як сервіс генерації природномовного пояснення на основі вже підготовлених даних.

## 9. Dashboard

Dashboard є центральною сторінкою користувача після авторизації.

Він об’єднує основні блоки застосунку:

* дані профілю користувача;
* інформацію про тренувальний план;
* стан виконання тренувань;
* показники харчування за день;
* добові цілі харчування;
* рекомендації;
* текстове AI-узагальнення рекомендацій.

Dashboard використовується для швидкого перегляду поточного стану користувача без необхідності переходити між усіма окремими розділами застосунку.

## 10. Модуль тренувань

Модуль тренувань забезпечує:

* перегляд бази вправ;
* фільтрацію вправ;
* формування персонального тренувального плану;
* перегляд тренувального плану по днях;
* позначення тренування як виконаного;
* збереження історії виконаних тренувань.

Генерація тренувального плану базується на даних профілю користувача, зокрема на рівні підготовки, цілі та бажаній кількості тренувань на тиждень.

## 11. Модуль харчування

Модуль харчування дозволяє користувачу вести базовий облік щоденного раціону.

Користувач може:

* переглядати добові цілі харчування;
* формувати або вручну редагувати добові цілі;
* додавати записи до журналу харчування;
* видаляти записи з журналу харчування;
* фіксувати калорії;
* фіксувати білки, жири та вуглеводи;
* фіксувати кількість води;
* переглядати підсумок харчування за день.

Рекомендації щодо харчування мають інформаційний характер. Вони не є медичними, дієтологічними або лікувальними призначеннями.

## 12. Логіка рекомендаційного модуля

Рекомендаційний модуль FitAI реалізовано як rule-based expert system.

Система аналізує:

* рівень підготовки користувача;
* ціль користувача;
* кількість тренувань у плані;
* кількість виконаних тренувань;
* відсоток виконання плану;
* суб’єктивну оцінку складності тренувань;
* добові цілі харчування;
* фактичні показники харчування за день;
* співвідношення фактичних показників із добовими цілями.

Приклади логіки правил:

* якщо користувач виконав менше половини запланованих тренувань, система може рекомендувати зменшити навантаження або переглянути регулярність занять;
* якщо користувач стабільно виконує більшу частину тренувального плану, система може рекомендувати поступове підвищення складності;
* якщо тренування часто позначаються як занадто складні, система може рекомендувати зменшити інтенсивність;
* якщо ціль користувача пов’язана зі зниженням ваги, система може акцентувати увагу на регулярності тренувань, кардіонавантаженні та контролі добових показників харчування;
* якщо ціль користувача пов’язана з набором м’язової маси, система може акцентувати увагу на силових вправах, поступовому збільшенні навантаження та достатності харчових показників;
* якщо фактична кількість води або макронутрієнтів суттєво відрізняється від добових цілей, система може сформувати інформаційну рекомендацію щодо коригування щоденного обліку.

Кожна рекомендація формується за правилом і може містити пояснення, чому саме вона була показана користувачу.

## 13. Інтеграція з OpenAI API

FitAI використовує OpenAI API як зовнішній AI-сервіс для генерації текстового узагальнення рекомендацій.

Загальна логіка роботи:

1. Backend отримує дані профілю користувача, тренувального плану, історії тренувань, харчування та рекомендацій.
2. Rule-based expert system формує структуровані рекомендації.
3. Backend передає підготовлений контекст до зовнішнього OpenAI API.
4. OpenAI API повертає текстове узагальнення.
5. Користувач бачить узагальнений текст у відповідному розділі застосунку або на Dashboard.

OpenAI API не використовується для зберігання даних користувача в базі FitAI та не є власною нейронною мережею проєкту. Основна логіка персоналізації залишається реалізованою на стороні FitAI через систему правил.

8. Згенерувати персональний тренувальний план.

9. Переглянути тренування по днях.

17. Перейти до сторінки рекомендацій.

18. Показати структуровані рекомендації та пояснення логіки їх формування.

19. Показати текстове AI-узагальнення рекомендацій, сформоване через OpenAI API.

20. Повернутися до Dashboard і показати зведення профілю, тренувань, харчування та рекомендацій.

## 16. Обмеження MVP

FitAI є дипломним MVP вебзастосунку та не є медичним сервісом.

Застосунок не замінює:

* консультацію лікаря;
* консультацію професійного тренера;
* консультацію дієтолога або нутриціолога;
* індивідуальну медичну діагностику;
* професійне призначення тренувальної або харчової програми.

Тренувальні та харчові рекомендації мають інформаційний і демонстраційний характер. Вони використовуються для показу алгоритмічної персоналізації в межах дипломного проєкту.

OpenAI API використовується тільки як зовнішній сервіс для генерації текстового узагальнення. FitAI не позиціонується як система з власною нейронною мережею.
